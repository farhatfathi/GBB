Yayasan Gerakan Baik Berdampak (No. AHU-0010854.AH.01.04.Tahun 2024) adalah yayasan beasiswa resmi yang menjalankan program Beasiswa Baik Berdampak (BBB) untuk mahasiswa aktif Universitas Diponegoro [[1]]. Didirikan oleh 10 alumni muda Undip, konsep utamanya adalah patungan donasi alumni — dengan motto #10AlumniBantuSatu, di mana alumni menyisihkan mulai dari Rp100.000/bulan untuk membiayai UKT atau penelitian akhir mahasiswa yang kesulitan finansial [[1]].

Programnya bukan sekadar uang. Beswan (penerima beasiswa) juga mendapat talkshow dan penugasan pengembangan diri [[1]][[2]]. Overview

FITUR UTAMA (MVP) PROGRAM
1. Database Beswan [Most complex #2]

Format database. Komponen: Metric-metric, database beswan, bar search & filter

Mengelola beswan: Filter, Upload CV, credential login portal beswan, Adding period beswan GBB (can be more than 1), dan very detail termasuk update kegiatan-kegiatan dia bisa dicek disini

Rapor: Ada absensi per event, refleksi (dari Portal Beswan) untuk setiap beswan (sheet database absensi terpisah, ada kolom event buat manggil)

2. Kurikulum & Library
Kurikulum: Kurikulum Beasiswa per periode

Dibagi per periode

Input Goal, Topik, Detail Topik, TORnya, per periode.

Kalau sudah terlaksana, link youtube & slide muncul

Library: Perpustakaan materi-materi bagus. soon be developed for RAG

Komponen: Metric-metric, database library, bar search & filter

Source: materi event, materi upload

Fitur Upload Materi, di summary AI, auto-tagging topic (bisa diedit jg) & masuk ke database,

3. Database Mentor

Format database. Komponen: Metric-metric, database mentor, bar search & filter

Mengelola Mentor: Filter, Upload CV, history ngisi event, hasil feedback form extracted per mentor

4. Event Talkshow: Mengelola Event dari awal-evaluasi based on Kurikulum & database mentor

Format database. Komponen: Metric-metric, database event, bar search & filter

Kasih alert untuk event selesai yang belum ada Link rekaman youtube & materi

5. Penugasan: Membuat tugas dari event yang terlaksana [Most complex #1]

Format database. Komponen: Metric-metric, database penugasan, database hasil penugasan, bar search & filter

Saat munculin penugasan, itu harus milih batch mana, setelh milih batch, user milih event, tapi bisa juga itu tugas yg non event

Penugasan dijabarkan flownya, ada penjelasan penggunaan fitur penugasan dari membuat sampai menilai, bisa ngasih feedback per beswan Ketika sudah ada hasil

Database penugasan: list tugas, kita udah pernah ngasi tugas apa aja. (Keliatan tagging kode eventnya & tgl nya) . Bisa juga bikin tugas not by event disini. Disetiap data, bisa dlihat siapa yang udah ngumpulin dan yang belum,

Database Hasil Penugasan; ini kalau jawaban udah disubmit beswan. Semuanya muncul disini. Kalau mau ngasih feedback dan menilai disini. (Keliatan tagging kode penugasan & tglnya)

Database Penugasan & Database Hasil Penugasan bisa kamu pasang dengan pola master-detail: klik 1 row tugas di tabel kiri/atas → expand detail hasil beswan di kanan/bawah. Jadi 2 database tetap ada tapi gak compete for attention.

6. Dashboard

Event : Kembangkan based on data

Growth : Kembangkan based on data

Beswan : Kembangkan based on data
